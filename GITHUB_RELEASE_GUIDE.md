# GitHub Release Creation Guide - ValidateJWT

## ?? Create GitHub Release (Easy Methods)

### **Method 1: Automated Script (Recommended)** ?

Just run:
```powershell
.\CreateGitHubRelease.ps1
```

**What it does:**
1. ? Checks Git status
2. ? Builds project in Release mode
3. ? Creates release package (zip file)
4. ? Creates and pushes Git tag
5. ? Creates GitHub Release (if gh CLI installed)
6. ? Uploads binaries as release assets

**Time:** ~1 minute

### **Method 2: Using GitHub CLI**

If you have GitHub CLI (`gh`) installed:

```powershell

# 2. Create release
```

```powershell

# Or download from https://cli.github.com/
```

### **Method 3: Manual via GitHub Web UI**

#### Step 1: Build Release Binaries
```powershell
# In Visual Studio
# Configuration > Release
# Build > Build Solution

# Or command line
msbuild ValidateJWT.csproj /p:Configuration=Release
```

#### Step 2: Create Zip Package
1. Create folder: `Release-v1.0.0`
2. Copy files:
   - `bin\Release\ValidateJWT.dll`
   - `bin\Release\ValidateJWT.xml`
   - `bin\Release\ValidateJWT.pdb`
   - `README.md`
   - `LICENSE.txt`
   - `CHANGELOG.md`
3. Zip the folder ? `ValidateJWT-v1.0.0.zip`

#### Step 3: Create Git Tag
```powershell
git tag -a v1.0.0 -m "ValidateJWT v1.0.0 - Initial Release"
git push origin v1.0.0
```

#### Step 4: Create Release on GitHub
1. Go to: https://github.com/johanhenningsson4-hash/ValidateJWT/releases/new
2. Fill in:
   - **Tag:** v1.0.0 (select from dropdown)
   - **Release title:** ValidateJWT v1.0.0 - Initial Release
   - **Description:** Copy content from `RELEASE_NOTES_v1.0.0.md`
3. Upload files:
   - Drag and drop `ValidateJWT-v1.0.0.zip`
   - Drag and drop `bin\Release\ValidateJWT.dll`
   - Drag and drop `bin\Release\ValidateJWT.xml`
4. Check ? **Set as the latest release**
5. Click **Publish release**

```

v1.0.1    - Patch release
v1.1.0    - Minor version (new features)
```

## ?? Troubleshooting

### Issue: Tag already exists
```powershell
# Delete local tag
git tag -d v1.0.0

# Delete remote tag
git push origin :refs/tags/v1.0.0

# Recreate tag
git tag -a v1.0.0 -m "Message"
git push origin v1.0.0
```

### Issue: Build fails
- Open Visual Studio
- Build > Clean Solution
- Build > Rebuild Solution in Release mode
- Check for errors

### Issue: GitHub CLI authentication
```powershell
# Login to GitHub CLI
gh auth login

# Check status
gh auth status
```

### Issue: Large file upload
GitHub has a 100 MB limit per file. If needed:
1. Use Git LFS for large files
2. Host large files elsewhere (Azure, AWS S3)
3. Link to external download in release notes
